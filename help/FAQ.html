What is Hack-It

Hack-It is a window manipulation program. It allows you to manipulate any window on your system in several different ways not normally available to you through the standard Windows user interface.

Back to Top

What do you mean? What sort of things can you do to windows?

The major functions you can perform on windows are:

  • Making a window visible or hidden
  • Causing a window to stay on top of others or not stay on top of others 
  • Adding an icon to the system tray for a window, and having that window minimize to the tray 
  • Manipulating process priorities to give a window’s process more CPU time 
  • Standard window functions such as restoring, minimizing, maximizing, and closing 
  • Forcibly terminating a process

Of the above functions, the first four are automatically added to the system menu of all top-level windows on your system for quick and easy access. The remaining functions are available through Hack-It’s interface. 

They all can be automated, matching a window’s title, and/or class name, and/or executable name to fire off any of the above functions, plus a couple additional ones:

  • Resize a window to a specific width and height
  • Move a window to a specific (x, y) position on the screen
  • Press a button on a with the specified text on a particular window
  • Perform any of the above functions to a named child window of the matched window

That sounds peachy and everything, but what good does it do me? Why should I use Hack-It?

The functions listed above are basic to every window, and yet the powers that be feel we should not be allowed to control them at will. Making a window stay on top, for example, is an extremely useful feature that you ordinarily cannot access. Imagine you are working on a full screen document, but wish to have a stock ticker or other informational window float above the active window so you can always check it at a glance. Without a utility like Hack-It, there is no way to make this happen. Similar cases can be made for the system tray. Windows will minimize to the system tray only of the developer of the application programmed it that way. This results in many applications that are always open, always cluttering your taskbar.

The ability to automate all of these functions makes the operations transparent. A window that you want to always float above others can be set up to do so. A window that you want to have minimize to the tray and hide itself when it starts can be set up to do you. Only your imagination limits what you can do.
